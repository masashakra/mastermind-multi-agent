Research Summary of 9 Major Papers

KEY FINDINGS
================================================================================

1. MASTERMINDEVAL PAPER
   Status: FOUND
   Paper: arxiv 2503.05891
   Rules location: Beginning of prompt, in natural language
   Rule format: Game rules (black/white peg feedback system)
   Rule length: ~100-150 words for comprehensive explanation
   Examples: Pre-played game scenarios shown as examples
   Repository: https://github.com/flairNLP/mastermind
   
   Key structure:
   - Game rules explained upfront (what are black/white pegs)
   - Example game history showing guesses and feedback
   - Task: Deduce the secret code from game history
   - Three evaluation modes: agentic, deductive (prompt-based), multiple-choice

2. TREE-OF-THOUGHTS (ToT) PAPER - GAME OF 24
   Status: FOUND (Exact prompts extracted)
   Paper: arxiv 2305.10601
   Repository: https://github.com/princeton-nlp/tree-of-thought-llm
   Rules location: Distributed across 5 different prompts
   
   Prompt phases:
   Phase 1 (Standard): "Use numbers and basic arithmetic operations (+ - * /) to obtain 24."
   Phase 2 (CoT): Rules + constraint about choosing two numbers per step
   Phase 3 (Propose): 5-shot examples showing format "A + B = C (left: remaining)"
   Phase 4 (Value): Evaluate thoughts as "sure/maybe/impossible"
   Phase 5 (Validate): Check final answer uses each number once
   
   Rule format: Natural language + structured format
   Rule length: 1-2 lines for core rules, 5-shot examples for format
   Performance: 4% (CoT baseline) -> 74% (ToT)
   File location: tot/prompts/game24.py

3. LOGIC-LM PAPER
   Status: FOUND
   Paper: arxiv 2305.12295
   Repository: https://github.com/teacherpeterpan/Logic-LLM
   Rules location: Distributed (problem statement + translation request + feedback)
   Rule format: Natural language + formal logic specifications
   Rule length: Problem-dependent (can be substantial)
   
   Three-component structure:
   1. Natural language problem (beginning)
   2. Symbolic translation request (middle - task instruction)
   3. Self-refinement loop (error-driven, iterative)
   
   Performance: 39.2% improvement over LLM alone, 18.4% over CoT
   Key innovation: Solver error messages drive prompt refinement

4. CHAIN-OF-THOUGHT (CoT) PROMPTING
   Status: FOUND (Exact examples extracted)
   Paper: arxiv 2201.11903
   Rules location: Implicit in examples (not stated explicitly)
   Rule format: Demonstration-based (show pattern, don't state rules)
   Rule length: Implicit (learned from examples)
   Examples: 8 optimal; 2-5 typical starting point
   
   Critical insight: "Language models are pattern-followers more than rule-followers"
   Key finding: "Do it like this" lands better than "follow these 15 requirements"
   
   Example structure:
   - Problem statement
   - Reasoning chain (intermediate steps)
   - Logical conclusion
   - Final answer

5. FEW-SHOT LEARNING FUNDAMENTALS
   Status: FOUND
   Paper: arxiv 2005.14165 (GPT-3)
   Rules location: Implicit in examples
   Rule format: K demonstrations at inference time (K = 10-100 typical)
   Examples: Optimal K = 8-10, diminishing returns beyond 10
   
   Three modes:
   - Zero-shot: No examples, baseline performance
   - One-shot: 1 example, modest improvement
   - Few-shot: K examples (K=8 optimal), significant improvement

6. CONSTRAINTLLM FRAMEWORK
   Status: FOUND
   Paper: arxiv 2510.05774
   Rules location: Distributed (role definition + examples)
   Rule format: Role-based + Python/pycsp3 code format
   Rule length: Brief role definition (~2 sentences) + code examples (20+ lines)
   Examples: 2-5 shot typical
   
   Three components:
   1. Problem specification (natural language)
   2. In-context examples (natural language + code)
   3. Target problem (to generate code for)

7. CONSTRAINT-COMPLIANT NETWORK OPTIMIZATION
   Status: FOUND
   Paper: arxiv 2509.07492
   Rules location: Multiple places (distributed throughout)
   Rule format: Six distinct components with different formats
   Rule length: 3-6 constraint sections, per-component variation
   
   Six components:
   1. Solution variable definition
   2. Constraint enforcement
   3. Linguistic constraints
   4. Context specifications
   5. Format requirements
   6. Validation criteria

8. RECAST: MULTI-CONSTRAINT INSTRUCTION FOLLOWING
   Status: FOUND
   Paper: arxiv 2505.19030
   Rules location: Explicit constraint list (middle section)
   Rule format: 19 different constraint types
   Rule length: Multiple constraints stacked
   
   Constraint types: Length, Content, Format, Negation, Style, Logical
   Key metric: HSR (Hard Constraint Satisfaction Rate) - must satisfy ALL
   Evaluation: Not partial credit; constraints are combined requirements

9. PROMPTING GUIDE RESOURCES
   Status: FOUND
   Website: https://www.promptingguide.ai/
   Best practices verified across papers:
   - Place examples near end of prompt
   - Keep examples in active context window
   - Maintain format consistency across examples
   - Use 2-8 examples (not 15+)

COMPARATIVE ANALYSIS
================================================================================

RULE LOCATION DISTRIBUTION:
- Beginning: ToT, MastermindEval, Logic-LM (problem), ConstraintLLM (role)
- Middle: Logic-LM (translation), Constraint Network, RECAST
- In context: ToT (propose), Logic-LM (feedback), ConstraintLLM (examples)
- In examples: CoT, Game 24 (5-shot), GPT-3 (K examples)

RULE FORMAT BY DOMAIN:
- Games (24, Mastermind): Natural language, simple and clear
- Logic problems: First-order logic + natural language
- Programming tasks: Python/code format + problem description
- Reasoning tasks: Structured chains-of-thought + examples
- Constraints: Distributed across multiple sections

RULE LENGTH PATTERNS:
- Simple game rules: 1-3 sentences
- Complex constraints: 3-6 distinct sections
- Logic specifications: Problem-dependent
- Examples: 2-5 shot typical, 8 shot optimal, 15+ degrades performance

NUMBER OF EXAMPLES:
- Minimum effective: 2-3 (shows pattern)
- Typical effective: 5-8
- Optimal: 8 (for CoT, Few-shot)
- Diminishing returns: Beyond 10
- Over-prompting: 15+ examples can degrade performance

================================================================================
KEY PATTERNS ACROSS ALL PAPERS
================================================================================

PATTERN 1: Multi-Phase Prompting
- Start with rules/problem statement
- Follow with examples/demonstrations
- End with task-specific instructions
- Result: More effective than single-phase approach

PATTERN 2: Distribution Works Better Than Concentration
- General rules at prompt beginning
- Specific rules near task location
- Context-specific rules embedded in examples
- Result: Better performance than rules-only section

PATTERN 3: Format Specification is Critical
- Exact output format matters more than rule statements
- Show format through examples (not description)
- Maintain identical structure across all examples
- Result: Consistent, predictable output

PATTERN 4: Examples > Explicit Rules
- Pattern-following beats rule-following
- "Show not tell" is more effective than "do this" instructions
- Implicit rules learned from examples beat explicit lists
- Result: 39-74% performance improvements documented

PATTERN 5: Feedback Loops Enable Refinement
- Use solver/validator output to refine responses
- Error messages guide next prompt iteration
- Neuro-symbolic integration improves reasoning
- Result: Significant improvements in complex domains

PATTERN 6: Constraint Languages Vary by Domain
- Games: Natural language rules
- Logic: Formal logic (FOL, CSP, SAT)
- Code: Programming language format
- Reasoning: Structured chain-of-thought

PATTERN 7: Task Decomposition Improves Performance
- Break complex tasks into evaluable steps
- ToT: 3 intermediate equations for Game of 24
- CoT: Step-by-step reasoning chains
- Logic-LM: Multi-turn translation + validation
- Result: Dramatically improved accuracy

================================================================================
CRITICAL INSIGHTS
================================================================================

INSIGHT 1: Rules Don't Stand Alone
Most successful papers combine rules with examples. Pure rule lists are less
effective. Best practice: Rule (2-3 sentences) + Examples (5-8 shot) + Task.

INSIGHT 2: Placement Matters
- Rules at beginning: Foundation
- Examples at end (before task): Most important for performance
- Tasks immediately after examples: Maintains context freshness

INSIGHT 3: Consistency is King
- All examples must have identical structure
- Format consistency matters more than rule statement clarity
- Inconsistent formatting degrades output quality

INSIGHT 4: Optimal Example Count
Fewer than 2: No pattern detected
2-5 examples: Shows pattern, reasonable performance
5-8 examples: Optimal for most tasks
8-10 examples: Near-optimal, slight gains
Beyond 10: Diminishing returns
Beyond 15: Potential performance degradation

INSIGHT 5: Game vs Logic vs Constraint Prompts Differ
- Games: Natural language rules, format examples
- Logic: Formal specifications, error feedback
- Constraints: Multi-component structure, validation criteria

INSIGHT 6: The "Show Don't Tell" Principle
Documented across all papers: LLMs are pattern followers.
Show examples instead of stating rules.
Format through demonstration beats rule statements.

INSIGHT 7: Error Messages Drive Refinement
Logic-LM's key innovation: Solver error messages feed back to prompt.
Iterative refinement using error feedback improves reasoning.
Neuro-symbolic approaches outperform pure-LLM by 39%+.

================================================================================
PRACTICAL RECOMMENDATIONS
================================================================================

FOR GAME-SOLVING PROMPTS (Mastermind, Game of 24):
1. Rule explanation (1-2 sentences, natural language)
2. Format specification showing expected output structure
3. 3-5 examples with same format as target task
4. Evaluation criteria (how to judge correctness)
5. Task specification (the actual game instance)

FOR CONSTRAINT REASONING PROMPTS:
1. Problem specification in natural language
2. Constraint list (clear enumeration)
3. Examples showing constraint application
4. Output format specification
5. Validation criteria
6. Target problem

FOR LOGICAL REASONING PROMPTS:
1. Natural language problem statement
2. Translation instruction (to formal representation)
3. Example translations (2-3 shot)
4. Feedback on previous attempt (if iterative)
5. Refinement request (if error)

FOR GENERAL COMPLEX TASKS:
1. Rule statement (1-3 sentences)
2. Format specification (show exact output)
3. Few-shot examples (5-8 typical)
4. Task specification (clear input/output)
5. Optional: Evaluation criteria
