WHAT IS COMPETITION?
================================================================================

Architecture:
  Shared Analysis + Shared Strategy + Competing Proposals

Workflow:
  1. Analyzer extracts constraints (once)
  2. Strategist proposes strategy (once)
  3. THREE proposers generate independently:
     - Conservative: Use tested colors, minimize risk
     - Aggressive: Test new colors, maximize information
     - Balanced: Mix both approaches
  4. Evaluator picks best based on:
     - Round number (early→aggressive, late→conservative)
     - Uniqueness score (color diversity)
     - Information potential
  5. Validator checks, submit

Expected:
  - More tokens (3 proposers = 2-2.5x)
  - Potentially better guesses (multiple perspectives)
  - Track which strategy works best

KEY METRICS TO TRACK
================================================================================

From Competition Results:
  1. Most Effective Proposer
     - Which strategy wins most rounds?
     - Conservative? Aggressive? Balanced?

  2. Win Rate Distribution
     - Does one dominate?
     - Or balanced mix?

  3. Performance
     - More guesses than Boss-Worker?
     - Same/better success rate?
     - Token cost worth it?

Example Analysis:
  If Aggressive wins 80%:
    → Early exploration is key
    → Favor new colors over tested
    → Use this learning for Coopetition

  If Balanced wins most:
    → Mix of strategies best
    → Neither extreme optimal
    → Balanced approach validated
